# [6.0.0](https://github.com/jenssimon/eslint-config-sfcc/compare/v5.3.0...v6.0.0) (2026-05-08)

- feat!: refine Rhino const handling with rhino-const-conflict rule ([631751c](https://github.com/jenssimon/eslint-config-sfcc/commit/631751c388cd9cbcadf141493f82ae1df9849027))

### BREAKING CHANGES

- rhino-const-compat no longer reports const in
  non-loop nested blocks (e.g. if/else, try/catch). The new
  rhino-const-conflict rule covers the unsafe subset of those cases.
  Projects that relied on rhino-const-compat to flag all nested-block
  const declarations will need to enable rhino-const-conflict, which
  is active in the recommended config by default.
